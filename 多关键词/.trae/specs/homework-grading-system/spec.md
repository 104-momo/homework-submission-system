# 小组作业提交与批改系统 Spec

## Why
当前缺乏一个便捷的在线平台，让学生能够提交小组作业，老师能够实时批改并管理成绩。需要一个简单易用的网页工具来提升作业提交和批改的效率。

## What Changes
- 创建用户身份认证系统，区分学生和老师角色
- 实现学生作业提交功能，支持文件上传和多个学生姓名输入
- 实现老师批改打分功能
- 实现成绩总表导出功能

## Impact
- 新建项目：小组作业提交与批改系统
- 技术栈：Next.js + TypeScript + Prisma + SQLite（或 PostgreSQL）

## ADDED Requirements

### Requirement: 用户身份认证
系统应提供用户身份认证功能，区分学生和老师角色。

#### Scenario: 学生登录
- **WHEN** 学生使用学生账号登录
- **THEN** 系统应显示学生界面，包含作业提交功能，不显示打分功能

#### Scenario: 老师登录
- **WHEN** 老师使用老师账号登录
- **THEN** 系统应显示老师界面，包含作业列表和打分功能

### Requirement: 学生作业提交
系统应允许学生上传作业附件并输入参与学生姓名。

#### Scenario: 单人作业提交
- **WHEN** 学生上传作业文件并输入一个学生姓名
- **THEN** 系统应保存作业文件和学生姓名信息

#### Scenario: 小组作业提交
- **WHEN** 学生上传作业文件并输入多个学生姓名（逗号分隔或换行）
- **THEN** 系统应保存作业文件和所有学生姓名信息

#### Scenario: 文件上传
- **WHEN** 学生选择文件并提交
- **THEN** 系统应支持常见文件格式（PDF、Word、图片等），文件大小限制为 10MB

### Requirement: 老师批改打分
系统应允许老师查看学生提交的作业并进行打分。

#### Scenario: 查看作业列表
- **WHEN** 老师登录后查看作业列表
- **THEN** 系统应显示所有已提交的作业，包括学生姓名、提交时间、作业文件链接

#### Scenario: 作业打分
- **WHEN** 老师查看某个作业并输入分数
- **THEN** 系统应保存分数，并标记该作业已批改

#### Scenario: 修改分数
- **WHEN** 老师需要修改已打分的作业
- **THEN** 系统应允许老师重新输入分数

### Requirement: 成绩总表导出
系统应允许老师导出成绩总表。

#### Scenario: 导出成绩表
- **WHEN** 老师点击导出成绩总表
- **THEN** 系统应生成包含所有学生姓名、作业信息、分数的表格，支持导出为 Excel 或 CSV 格式
